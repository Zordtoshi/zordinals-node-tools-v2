Zordinals — On-Chain Inscriptions for Zcash

Zordinals bring ordinal theory to Zcash — allowing you to inscribe images, text, files, and arbitrary digital artifacts directly onto the Zcash blockchain, permanently, immutably, and without relying on off-chain storage.

Think of them as NFTs with actual teeth: no servers, no IPFS links that disappear, no “metadata gateways,” no rug pulls via deleted images. Just raw, verifiable, forever-living data baked into the chain.

What Are Zordinals?

Zordinals apply Bitcoin’s ordinal theory to Zcash’s transparent UTXO model:

  • Every satoshi-like unit of ZEC inside a UTXO can be uniquely numbered.
  • Those individually trackable units can be assigned data (“inscriptions”).
  • Once inscribed, the data becomes a permanent part of the blockchain, because it's committed inside the raw transaction.
  • Ownership of a Zordinal = ownership of the UTXO that contains the inscribed satoshis.

The result?

  • • Fully on-chain NFTs
  • • Permanently stored
  • • Never broken by external links
  • • No centralized hosts
  • • No IPFS pinning required

If an “NFT” calls to off-chain data: It’s not a Zord. It’s just… a JPEG with commitment issues.

Immutability & On-Chain Guarantees

Zordinals data is inserted directly into a Zcash raw transaction. This means:

  • • It is hashed into the block.
  • • It is validated by every node.
  • • It can’t be changed, removed, or overwritten.
  • • It does not depend on any third-party storage.

As long as Zcash exists, your Zordinals exist.

How Zordinals Store Data (Simple Explanation)

Zcash has limits on how much data can fit into a single transaction field, so inscriptions use a clever trick:

  • 1. Your data (image/text/etc) is chunked — It is split into multiple small pieces.
  • 2. Each chunk is encoded — We use a lightweight, compact Base64-like encoding for safe insertion.
  • 3. Chunks are written into specific locations inside one or more UTXOs — Each chunk is stored inside the ScriptSig / script fields of raw transactions.
  • 4. “Link markers” identify the sequence — Each inscription has markers that indicate: Start of inscription, Middle chunks, End of inscription, What order they belong in, What inscription ID they are part of. Think of it like LEGO blocks with matching stickers.
  • 5. Indexers can reconstruct the inscription perfectly — By scanning the chain, reading the chunks in order, and reassembling them.

Nothing lives off-chain. Everything needed is inside the blockchain itself.

How a Zordinal Is Created (High-Level)

A special P2SH output is constructed to hold the inscription marker.

The inscription data (chunked) is embedded into ScriptSigs across one or more inputs.

A funding transaction (“TX0”) prepares the structure.

A reveal transaction publishes the completed inscription on-chain.

From that point forward: The UTXO becomes the “holder” of the Zordinal. Sending the Zordinal is simply spending that specific UTXO.

Why Zordinals?
  • • Permanence — Zordinals never disappear! Even if every server and IPFS gateway went offline.
  • • Verifiability — Every byte is stored on-chain and validated by consensus rules.
  • • Independence — No dependency on centralized hosting.

  • • Transparency — Since inscriptions rely on Zcash's transparent UTXOs (not shielded), they are easy to scan, index, and verify.
What Counts as a “Real” Zordinal?

A genuine Zordinal must:

  • • Store 100% of its data ON-CHAIN
  • • Use valid Zordinals encoding and markers
  • • Not rely on external resources (IPFS, HTTP URLs etc.)
  • • Follow the sequential inscription data chunk structure

If it points off-chain? It’s not a Zordinal. It’s cosplay.
