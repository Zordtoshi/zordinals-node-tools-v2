What this is

These pages are a small toolkit for Zcash full node runners. They let you:

  • • View Zimmutable Zordinals by inscription TXID.
  • • Explore all content in your local /content folder.
  • • Monitor node status, peers, mempool and wallet balances.
  • • Control your wallet with UTXO-aware sends and multi-wallet use.
  • • Inspect raw transactions and run CLI-style node commands.
  • • Build, convert and repair collection metadata for Zords.
  • • Define pixel-perfect “fingerprints” for traits using a 20×20 grid.

Pages overview
  • Info – this page. High-level overview, quick usage and some basic wallet tips.
  • ZNode Status – overview of chain height, peers, mempool, wallet balances, UTXOs and wallet transactions.
    Supports importing keys, UTXO-aware sends and TX status checks.
  • CLI Controls – dev console that wraps common zcash-cli RPC commands with forms, helper text and raw JSON output.
  • Zordinals Viewer – enter an inscription TXID (with i0 suffix).
    If the content exists in /content, it loads locally; otherwise the node is queried and the content is decoded and cached.
  • Explore Zords – grid view of all files in your /content directory. Click any tile for a larger modal view, JSON overlays, copy TXID, copy image or download.
  • Metadata Tools – build collection metadata from scratch, edit existing JSON, split/merge files, and convert HashLips _metadata.json into Zord-style per-token files plus a master index.
  • Define Traits – 20×20 pixel grid for defining trait “fingerprints” per layer/value. Outputs JSON that can be used by indexers, explorers or off-chain tools.
  • Zordinal Theory – a deeper write-up of how Zordinals work, why everything stays on-chain, and what counts as a “real” Zord.
Metadata & traits workflow

The Metadata Tools and Define Traits pages are designed to work together when building a collection:

  • • Use Metadata Tools → BASIC to define your collection name, description, supply, and base fields, then generate clean per-token JSON and a master.json index.
  • • Use ADVANCED to tweak attributes, merge/split metadata files, fix IDs, or attach inscription IDs for images you have already inscribed.
  • • Use HASHLIPS ➜ Zordinals to drop in a HashLips _metadata.json, convert it into Zord-style per-token files, and then map each token to an inscription ID.
  • • On Define Traits, paint the 20×20 grid to mark where pixels appear for a given trait (e.g. “Hat: Red Cap”). Each fingerprint is saved as JSON so indexers/frontends can instantly tell if those pixels exist in any image.

Together, these pages give you an on-chain image, a clean JSON metadata layer, and machine-readable trait fingerprints – all built locally alongside your Zcash node.

Quick usage
  • 1. Run a fully synced zcashd full node with RPC enabled.
  • 2. Set up your .env pointing NODE_RPC_URL at your node (e.g. http://127.0.0.1:8232).
  • 3. Start viewer.js (the web server for these tools).
  • 4. Open http://localhost:4000 in a browser to begin.
Sending & receiving basics
  • • Use ZNode Status → Wallet Balances to see your current wallets and UTXOs. Tick UTXOs you want to spend.
  • • Click SEND ZEC, pick a wallet, enter an amount and destination address, and confirm. The app keeps 0.0001 ZEC aside for fees.
  • • After sending, use CHECK STATUS to monitor confirmations for that TX.
  • • To receive, simply share your transparent address, then watch the wallet + UTXO panels update as blocks confirm.
